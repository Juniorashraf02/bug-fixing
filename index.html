Question 01: How does JavaScript work?

Answer: JavaScript is a lightweight and a scripting language that is used to create animated or interactive contents in websites and web-applications. JavaScript makes a website live. JavaScript is a case sensitive language. All identifiers of javascript are case sensitive. In JavaScript the source codes are transformed into an abstract syntax tree, a data structure that reprensts code in a format that machines can deal with it. JavaScript reads the codes line by line. So we have to arrange the codes carefully. Sometimes the unarranged line could create problems. Conceptually, JavaScript is much like HTML text which is transformed into its Document Object Model reprenstation, which is why code actually works. After creating the HTML & css file, the javaScript file is linked. And whatever codes are written, browser shows that activity.

Question 02: Local storage vs Session storage

Answer: Local storage and session storage are two objects. LocalStorage and SessionStorage are part of the web storage API. These are the two great tools for saving a key or value pairs locally. LocalStorage and SessionStorage have the same API. The main difference in them is that with sessionStorage, the data is persisted only until the window or tab is closed. But in case of Local storage the data are saved in it even we close the browser and reopen it we can see that the data is saved in our local storage. Even if we open it after a month or a year we can see that data is still saved to our local storage! Local storage capacity depends on the browser and in case of session storage it is 5Mb. If we create multiple sessions storage, for each tab a session storage will be created but the local storage will be the same!
